## Appointment Flow

arg | user dashboard | system proccess

- user adds appointment | user sees pending
- user clicks cancel | request moved
  end

- admin clicks accept | user sees proccessing | request moved to appointed dashboard
  - admin clicks cancel | user sees canceled and the reason for cancelation
  - admin clicks done | user sees the full detail of its completed proccess on history
    - end

- admin clicks decline | user sees declined
 - end
